# Contributing to ML.NET Network Anomaly Detection

First off, thank you for considering contributing to this project! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue using the **Bug Report** template. Include as much detail as possible, such as your environment, steps to reproduce, and any logs from the service.

### Suggesting Enhancements
Have an idea for a new feature? We'd love to hear it! Open an issue using the **Feature Request** template.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code lints.
5. Issue that pull request!

## Style Guide

### Coding Standards
- Follow standard C# naming conventions (PascalCase for classes/methods, camelCase for local variables).
- Use meaningful variable names.
- Keep methods focused and concise.

### Commit Messages
- Use the imperative mood ("Add feature" not "Added feature").
- Keep the first line under 50 characters.
- Reference issues and pull requests liberally.

## Documentation
Update the `README.md` or other documentation files if your changes affect how a user interacts with the application.

## Versioning
This project follows [Semantic Versioning](https://semver.org/). Maintainers will handle official version bumps.

Thank you for your contributions!
